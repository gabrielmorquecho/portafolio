SQL injection vulnerability in WHERE clause allowing retrieval of hidden data

Laboratorio 01: SQL injection in WHERE clause

Progreso SQLi: 1 de 18 laboratorios completados.
Detalles Técnicos
  • Plataforma: PortSwigger Academy
  • Vulnerabilidad: SQLi (In-band)
  • Herramienta: Burp Suite
Descripción

Manipulación de la lógica SQL para anular la condición AND released = 1 y extraer registros ocultos del sistema.

01. Análisis de Parámetro

Identificamos el parámetro category como el vector de entrada vulnerable.

[EVIDENCIA: PETICIÓN ORIGINAL]

02. Explotación (Payload)

Inyectamos el payload ' OR 1=1-- para forzar la condición verdadera en la base de datos.

GET /filter?category=Gifts'+OR+1=1-- HTTP/2

[EVIDENCIA: MODIFICACIÓN EN REPEATER]

03. Resolución y Verificación

El sistema responde mostrando los productos ocultos (released = 0).

[EVIDENCIA: PANTALLA SOLUCIONADA]
