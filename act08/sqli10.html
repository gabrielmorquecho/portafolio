Laboratorio 10: SQL Injection UNION Attack – Retrieving Multiple Values in a Single Column

Progreso SQLi: 10 de 18 laboratorios completados.
Detalles Técnicos
  • Tipo: UNION-based SQL Injection
  • Técnica: Data Concatenation
  • Tabla objetivo: users
  • Columnas: username, password
  • Objetivo: Extraer credenciales en una sola columna
Análisis del Ataque

En algunos casos la aplicación web solo muestra una columna con datos de texto. Esto impide recuperar múltiples valores directamente con UNION SELECT. Para solucionar esto se utiliza una técnica de concatenación de datos, uniendo varias columnas dentro de un mismo campo utilizando operadores como ||.

01. Interceptar la Solicitud

Primero abrimos el laboratorio utilizando el navegador de Burp Suite. Activamos la opción Proxy → Intercept ON para capturar la solicitud HTTP cuando seleccionamos una categoría de productos.

[EVIDENCIA: INTERCEPCIÓN DE REQUEST]

02. Enviar la Solicitud a Repeater

Una vez capturada la petición, la enviamos al módulo Repeater presionando Ctrl + R para poder modificar manualmente el parámetro vulnerable category.

[EVIDENCIA: REQUEST EN REPEATER]

03. Determinar Columnas y Tipo de Datos

Primero debemos verificar cuántas columnas devuelve la consulta original y cuáles contienen datos de tipo texto.

'+UNION+SELECT+NULL,'abc'--

La respuesta confirma que la consulta devuelve dos columnas, pero solo una de ellas acepta datos de texto.

04. Concatenación de Valores

Para poder mostrar el nombre de usuario y la contraseña en una sola columna utilizamos el operador de concatenación ||. También agregamos un separador ~ para distinguir ambos valores.

'+UNION+SELECT+NULL,username||'~'||password+FROM+users--

Esto permite que ambos valores aparezcan dentro de la misma columna de resultados.

[EVIDENCIA: DATOS CONCATENADOS]

05. Identificación de Credenciales

En la respuesta de la aplicación aparecen los registros concatenados, por ejemplo:

  • administrator~contraseña
  • wiener~contraseña
  • carlos~contraseña

Se copia la contraseña del usuario administrator para iniciar sesión en la aplicación.

06. Resultado: Acceso a la Cuenta Administrator

Finalmente accedemos a la sección Login e introducimos las credenciales obtenidas. Al iniciar sesión correctamente como administrator se completa el laboratorio.

[EVIDENCIA: LOGIN ADMINISTRADOR]

Mitigación Recomendada

Para prevenir ataques de SQL Injection es fundamental utilizar consultas parametrizadas y validar las entradas del usuario antes de enviarlas a la base de datos.

Incorrecto (Vulnerable): query = "SELECT * FROM products WHERE category = '" + user_input + "'"; Correcto (Seguro): PreparedStatement pstmt = connection.prepareStatement( "SELECT * FROM products WHERE category = ?"); pstmt.setString(1, user_input); ResultSet results = pstmt.executeQuery();
