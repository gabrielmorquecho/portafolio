Notas de Seguridad: SQL Injection (SQLi)

¿Qué es SQL Injection?

Es una vulnerabilidad crítica donde las aplicaciones web incorporan directamente entradas del usuario en consultas SQL sin sanitización. Esto permite a un atacante insertar código malicioso para alterar consultas y acceder a datos no autorizados.

Crítico (OWASP, 2021)

In-band (Clásico)

Se usa el mismo canal para el ataque y la respuesta.

  • Error-based: Aprovecha mensajes del sistema.
  • Union-based: Combina resultados legítimos con extraídos.

Blind (Ciego)

Inferencia de datos sin mensajes de error visibles.

  • Boolean-based: Respuestas de verdadero/falso.
  • Time-based: Retrasos intencionales (SLEEP).

Out-of-band

Exfiltración a través de canales externos (DNS/HTTP).

Se utiliza cuando el canal principal está restringido.

Impacto en la Tríada CIA

  • Confidencialidad: Extracción de credenciales.
  • Integridad: Modificación de registros.
  • Disponibilidad: Eliminación de tablas o DoS.

Principios de Defensa

La medida más efectiva es el uso de Consultas Parametrizadas (Prepared Statements).

  • Principio de mínimo privilegio.
  • Validación mediante listas blancas.
