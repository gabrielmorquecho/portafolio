Análisis de Intrusiones con el Modelo Diamante

El Modelo Diamante de Análisis de Intrusiones es una metodología utilizada en ciberseguridad para analizar ataques informáticos mediante la relación entre cuatro elementos principales: adversario, capacidad, infraestructura y víctima. Combinado con la Cyber Kill Chain, permite identificar patrones de comportamiento, correlacionar eventos y comprender el desarrollo completo de una intrusión.

Parte 1: Los Cuatro Nodos del Modelo Diamante

Adversario

El actor o grupo responsable del ataque.

Puede ser un individuo, una nación-estado o un grupo criminal. Ej: Grupo APT28 (Fancy Bear) o un empleado deshonesto.

Capacidad

Las herramientas, técnicas y procedimientos (TTPs).

Recursos que el adversario utiliza para ejecutar el evento. Ej: Ransomware, exploit de día cero o phishing.

Infraestructura

Los recursos físicos o lógicos del adversario.

Usados para entregar la capacidad o comunicarse. Ej: Servidores C2, IPs, dominios maliciosos, proxies.

Víctima

El objetivo del ataque.

Puede ser una organización, persona, dirección IP o activo específico. Ej: Servidor de BD de un banco o correo de un directivo.

Parte 2: Análisis del Evento

Escenario: Una organización detecta comportamiento anómalo en un equipo. El análisis revela: detección de malware con dominios C2 embebidos, resolución a IPs externas, múltiples hosts conectándose a esas IPs, y registros WHOIS que revelan el posible origen del atacante.

Metacaracterística Descripción del Evento
Marca de tiempo Momento en que se detecta el malware y las conexiones sospechosas hacia IP externas.
Fase Compromiso inicial, ejecución del malware y comunicación con C2.
Resultado El equipo fue comprometido y estableció comunicación con infraestructura externa del atacante.
Dirección Víctima → Infraestructura. Tráfico saliente desde los hosts internos hacia dominios e IP externas.
Metodología Uso de malware con capacidades de comando y control (C2), resolución DNS y conexiones remotas.
Recursos Malware, dominios C2, direcciones IP externas, logs de red y registros WHOIS/IP.
Preguntas del Escenario
1. ¿Quién es el adversario?

Un atacante o grupo cibercriminal externo que opera el malware y la infraestructura C2. Su posible origen puede identificarse mediante análisis WHOIS y rastreo de IPs.

2. ¿Cuál es la capacidad utilizada?

Malware con funciones de comunicación C2, diseñado para conectarse a dominios específicos y permitir el control remoto. Posiblemente distribuido mediante phishing.

3. ¿Qué tipo de infraestructura se emplea?

Infraestructura externa compuesta por dominios maliciosos, servidores C2 y direcciones IP públicas para la comunicación con los hosts comprometidos.

4. ¿Quién es la víctima primaria?

El equipo o usuario de la organización donde inicialmente se detectó el comportamiento anómalo y el malware.

5. ¿Existe evidencia de movimiento lateral?

Sí. Los logs muestran múltiples hosts conectándose a las mismas IPs externas, lo que sugiere posible propagación del malware o movimiento lateral dentro de la red interna.

Parte 3: Relación con la Cyber Kill Chain

Entrega (Delivery) Envío de Phishing

El atacante envía un correo de phishing dirigido al administrador con un archivo o enlace malicioso adjunto.

Explotación Ejecución de Malware

Al abrir el archivo malicioso, el malware se ejecuta en el equipo comprometido aprovechando una vulnerabilidad del sistema.

Instalación / Acciones sobre el objetivo Detección de Malware

El malware se instala de forma persistente en el sistema comprometido, listo para ejecutar acciones sobre el objetivo.

Comando y Control (C2) Conexión a C2

El malware establece comunicación con el servidor C2 del atacante, permitiendo el acceso y control remoto total del host.

Parte 4: Hilos de Actividad (Pivoting)

Hilo 1 – Víctima 1
1 El atacante envía un correo de phishing dirigido al administrador.
2 El administrador abre el archivo o enlace malicioso.
3 Se ejecuta el malware en el equipo comprometido.
4 El malware establece conexión con el servidor C2.
5 El atacante obtiene acceso remoto al host.
Hilo 2 – Víctima 2
1 El atacante utiliza el host comprometido como proxy o punto de acceso interno.
2 Se realizan conexiones hacia otra máquina dentro de la red.
3 El malware o las credenciales robadas permiten comprometer una segunda víctima.
4 La segunda víctima también comienza comunicación con la infraestructura C2.
Relación entre ambos: Pivoting

El pivoting ocurre cuando el atacante utiliza el primer equipo comprometido para desplazarse lateralmente y atacar otros sistemas dentro de la red. La Víctima 1 funciona como puente para comprometer a la Víctima 2, extendiendo el alcance del ataque sin necesidad de un nuevo vector de entrada externo.

Conclusión

El análisis realizado mediante el Modelo Diamante permitió identificar la relación entre el atacante, las capacidades utilizadas, la infraestructura maliciosa y las víctimas afectadas. La Cyber Kill Chain facilitó comprender las diferentes fases del ataque y detectar indicios de movimiento lateral dentro de la red.

El estudio de los hilos de actividad demostró cómo un host comprometido puede utilizarse para realizar pivoting y expandir la intrusión hacia otros sistemas. Este tipo de metodologías son fundamentales para mejorar la detección, análisis y respuesta ante incidentes de ciberseguridad.

Metodologías Aplicadas:
  • Modelo Diamante: Correlación entre adversario, capacidad, infraestructura y víctima.
  • Cyber Kill Chain: Identificación de cada fase del ataque desde la entrega hasta el control C2.
  • Pivoting / Hilos: Análisis de movimiento lateral y propagación dentro de la red interna.

"El Modelo Diamante no solo describe un ataque; revela la inteligencia detrás de él, permitiendo anticipar al adversario en lugar de solo reaccionar."
