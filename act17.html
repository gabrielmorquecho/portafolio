Evaluación de Vulnerabilidades con CVSS v3.1

En el ámbito de la ciberseguridad, no basta con detectar una falla técnica: es indispensable cuantificar su gravedad de manera objetiva y reproducible, de modo que los equipos de seguridad puedan priorizar sus acciones de mitigación con base en criterios técnicos sólidos. Esta actividad aplica el modelo CVSS v3.1 sobre un escenario real para construir correctamente una cadena vector, calcular la puntuación base e interpretar su significado organizacional.

Marco Teórico: Métricas CVSS v3.1

El Common Vulnerability Scoring System (CVSS) v3.1 es un estándar para medir la gravedad de vulnerabilidades de seguridad. La puntuación base se obtiene a partir de dos subgrupos: métricas de explotabilidad y métricas de impacto.

Métricas de Explotabilidad

Attack Vector (AV)

Contexto desde el cual es posible explotar la vulnerabilidad. Network · Adjacent · Local · Physical.

Attack Complexity (AC)

Condiciones técnicas adicionales necesarias para la explotación. Low: sin condiciones especiales. High: condiciones difíciles de reproducir.

Privileges Required (PR)

Nivel de acceso autenticado requerido. None · Low (usuario básico) · High (administrador).

User Interaction (UI)

Si la explotación requiere participación activa de otro usuario. None: sin intervención. Required: el usuario debe actuar.

Métricas de Impacto

Scope (S)

Unchanged: impacto limitado al componente. Changed: afecta otros sistemas.

Confidentiality (C)

None · Low: acceso limitado · High: divulgación total.

Integrity (I)

None · Low: modificación limitada · High: datos totalmente alterados.

Availability (A)

None · Low: interrupciones parciales · High: servicio inaccesible.

Escenario

Una organización detecta una vulnerabilidad en un sistema web interno con las siguientes características:

  • Puede explotarse a través de la red.
  • La explotación requiere baja complejidad.
  • Se necesitan privilegios elevados.
  • No requiere interacción del usuario.
  • No cambia el alcance del sistema afectado.
  • Impacto en Confidencialidad, Integridad y Disponibilidad: Bajo.

Métrica Valor Justificación
AV – Attack Vector N La vulnerabilidad puede explotarse a través de la red de forma remota.
AC – Attack Complexity L La explotación requiere baja complejidad; no se necesitan condiciones especiales.
PR – Privileges Required H Se necesitan privilegios elevados previos para ejecutar el ataque.
UI – User Interaction N No requiere ninguna interacción activa de un usuario legítimo.
S – Scope U No cambia el alcance; el impacto se limita al componente vulnerable.
C – Confidentiality L El impacto en confidencialidad es bajo; no se comprometen datos críticos.
I – Integrity L El impacto en integridad es bajo; solo modificaciones limitadas posibles.
A – Availability L El impacto en disponibilidad es bajo; interrupciones menores al servicio.

Interpretación Técnica

La puntuación final de 4.7 sitúa esta vulnerabilidad en el rango de Severidad Media. Aunque el ataque es sencillo de ejecutar y puede realizarse de forma remota, el requisito de privilegios elevados reduce significativamente la cantidad de atacantes capaces de explotarla.

AV:N — Vector de Ataque

La vulnerabilidad es explotable de forma remota a través de la red; el atacante no requiere acceso físico ni local al servidor. Esto aumenta el nivel de exposición del sistema.

AC:L — Complejidad de Ataque

No existen condiciones especiales para que el ataque tenga éxito. Una vez con los privilegios requeridos, la explotación puede ejecutarse de forma directa y repetible.

PR:H — Privilegios Requeridos

Este es el principal factor que reduce la severidad. El atacante necesita previamente una cuenta con permisos elevados, lo que limita drásticamente el número de posibles actores capaces de explotar la falla.

UI:N — Interacción del Usuario

La explotación no requiere que un usuario legítimo realice ninguna acción. Esto facilita que el ataque pueda ejecutarse de manera autónoma.

C:L / I:L / A:L — Impacto Triple Bajo

La vulnerabilidad puede afectar parcialmente la información, modificar datos de forma limitada o causar afectaciones menores al servicio, pero no compromete completamente el sistema.

Reflexión Técnica: Priorización de Riesgos con CVSS

El ejercicio con CVSS v3.1 demuestra que la gravedad de una vulnerabilidad no depende de un solo factor, sino de la interacción entre todos sus parámetros. En este caso, aunque el vector de red y la baja complejidad elevarían el riesgo de forma aislada, el requisito de privilegios elevados actúa como un control natural que reduce la puntuación final a severidad media.

Para un equipo de seguridad, esto implica que la prioridad de remediación debe centrarse en proteger las cuentas privilegiadas: si un atacante obtiene esas credenciales, la barrera de entrada desaparece y la vulnerabilidad se vuelve inmediatamente explotable desde cualquier punto de la red.

Pilares del Análisis CVSS:
  • Objetividad: Puntuación numérica comparable entre distintas vulnerabilidades y equipos.
  • Contexto: Cada métrica refleja condiciones reales del entorno, no supuestos genéricos.
  • Priorización: Permite enfocarse en las vulnerabilidades de mayor impacto operacional antes que en las más numerosas.

"CVSS no mide el peligro en abstracto; mide el peligro en contexto. Y el contexto es todo en ciberseguridad."
